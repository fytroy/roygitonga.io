Automating Daily Health Checks: The Daily System Health Reporter

Published on June 25, 2025 | Category: Automation, Windows, System Admin | By Rodney Gitonga

For IT administrators and even keen individual users, having a clear and consistent overview of system health is paramount. Manual checks are time-consuming and prone to human error. This led me to develop the "Daily System Health Reporter," a Python script designed to automate system monitoring, generate a clean HTML report, and deliver it directly to your inbox.

The Need for Automation

Imagine having multiple Windows servers or workstations to manage. Logging into each one daily to check disk space, CPU usage, service status, and event logs is simply inefficient. This script was created to consolidate these checks into an automated, digestible report, allowing for proactive issue detection.

What the Reporter Gathers:

  • System Information: Hostname, OS version, boot time.
  • CPU Usage: Current and average CPU load.
  • Memory Usage: Total, used, and available RAM.
  • Disk Usage: Breakdown of all logical drives, showing free space.
  • Network Status: IP address, network connections, and transfer statistics.
  • Running Services: Lists critical services and their status.
  • Key Event Log Entries: Filters and presents important warnings/errors from the Windows Event Log.
  • Installed Applications: (Optional) List of recently installed software.

How It Works (Technical Breakdown):

The script leverages Python's powerful `psutil` library for cross-platform system information. For Windows-specific data, especially for fetching details like Event Log entries or installed applications more robustly, I explored using `WMI (Windows Management Instrumentation)` through Python libraries like `wmi` (though `subprocess` calls to `wmic` or PowerShell can also be used).

The generated report is an HTML file. Python's f-strings and basic HTML structuring allow for dynamic content insertion. This makes the report easily readable in any web browser or email client.

Sending the email is handled by Python's `smtplib` and `email.mime` modules, allowing for robust email construction (including attachments for the HTML report) and secure transmission (using SSL/TLS).

Setting it Up for Daily Runs:

The beauty of this script lies in its automation. It's designed to be scheduled using Windows Task Scheduler. You simply configure it to run daily at a specific time, and it handles the rest â€“ gathering data, generating the report, and emailing it.

Impact and Future Outlook:

This project significantly reduces the manual effort required for daily system health checks, freeing up time for more critical tasks. It also provides a consistent record of system performance, which is invaluable for long-term trend analysis and troubleshooting. Future enhancements could include graphical representations within the HTML report, integration with cloud-based logging services, or even predictive analytics based on historical data.
